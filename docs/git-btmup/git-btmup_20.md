# 结论

> 原文：[`jwiegley.github.io/git-from-the-bottom-up/5-Conclusion.html`](http://jwiegley.github.io/git-from-the-bottom-up/5-Conclusion.html)

这些年来，我使用过许多版本控制系统和备份方案。它们都提供了检索文件过去内容的功能。大多数都有显示文件随时间变化的方式。许多允许你回到过去，开始一条不同的推理线，然后稍后把这些新想法带回到现在。然而，更少的有精细控制这个过程的能力，让你能够以你认为最好的方式收集你的想法，向公众展示你的观点。Git 让你能够做所有这些事情，而且相对容易——一旦你理解了它的基础。

它不是唯一拥有这种能力的系统，也不是总是使用最好的界面来展示其概念。然而，它确实有一个坚实的基础来工作。在未来，我想会有许多新方法被发明出来，以利用 Git 允许的灵活性。大多数其他系统都让我相信它们已经达到了概念上的顶峰——从现在开始，所有其他的东西都将是之前所见的缓慢改进。然而，Git 给我的是相反的印象。我感觉我们只是刚刚开始看到它那看似简单的设计所承诺的潜力。
